How to Submit Applications to FlatFree

Back to Blog · June 10, 2026 by Natalie

Submitting your application to FlatFree is designed to be straightforward. Our entire submission process is pull-request-based, with automated CI that validates, builds, and scans your application. No human gatekeepers, no subjective reviews — just automated checks.

Here's the complete step-by-step guide.

Step 1: Fork the Repository

Start by forking the FlatFree repository on GitHub. This gives you your own copy of the repository where you can make changes without affecting the main branch.

git clone https://github.com/YOUR_USERNAME/FlatFree.git
cd FlatFree

Step 2: Create Your App Directory

Applications live under the flatpak/ directory, organized by their reverse-domain-name identifier. For example, if your app is com.example.MyApp, create:

mkdir -p flatpak/com/example/MyApp

The directory structure mirrors the application ID, which ensures uniqueness and follows Flatpak conventions.

Step 3: Write a Flatpak Builder Manifest

Inside your app directory, create a com.example.MyApp.yml file. This is your flatpak-builder manifest that describes how to build and package your application. At minimum, it should include:

  • The application ID matching your directory name
  • The runtime and SDK (typically org.freedesktop.Platform and org.freedesktop.Sdk)
  • The build system and build commands
  • Sources — either a tarball URL, git repository, or local path
  • Any required modules or dependencies
  • Finish arguments for sandbox permissions

For examples, look at the existing apps in the flatpak/ directory. The com.acreetionos.MediaWriter app is a good reference.

Step 4: Open a Pull Request

Once your manifest is ready, commit your changes and push to your fork:

git add flatpak/com/example/MyApp/
git commit -m "Add com.example.MyApp"
git push origin main

Then open a pull request on the main FlatFree repository. Our CI pipeline will automatically trigger.

Step 5: CI Validates and Builds

When you open a PR, our CI pipeline runs the following checks automatically:

  • Manifest validation — checks that your YAML is well-formed and follows the required schema
  • Flatpak builder build — actually builds your application in a clean environment to verify it compiles
  • Malware scan — runs security analysis on the built application
  • License check — verifies the application uses an approved free software license
  • Export test — confirms the application exports properly to a Flatpak bundle

If all checks pass, a maintainer will merge your PR. Your application is now part of FlatFree!

What Happens After Merge

Once merged, our deployment pipeline handles the rest:

  • The application is built and added to the Flatpak repository
  • The metadata is updated on the website
  • The app appears on the apps page within minutes
  • Users can install it with flatpak install --user flatfree com.example.MyApp

You don't need to do anything else. Updates to your manifest will automatically trigger rebuilds.

Tips for a Smooth Submission

  • Test your manifest locally with flatpak-builder --force-clean --repo=repo build-dir com.example.MyApp.yml before submitting
  • Keep your sandbox permissions minimal — only request what your app actually needs
  • Use the latest runtime version available
  • Include a metainfo.xml or appdata.xml file for proper metadata display
  • If you get stuck, open a discussion on GitHub or check the docs

We're excited to see what you build. Submit your application today and be part of the open alternative.
